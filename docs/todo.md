# Obsidian NWT Linker 実装タスクリスト

## Phase 1: コア変換

### 1.1 データ定義

- [ ] デフォルト設定オブジェクトを定義する
- [ ] デフォルト書名エイリアステーブルを定義する
- [ ] `ScriptureReference` 型を定義する
- [ ] `ScripturePart` 型を定義する
- [ ] URLテンプレートのデフォルト値を定義する

### 1.2 正規化処理

- [ ] 全角数字を半角数字へ変換する関数を実装する
- [ ] 全角コロン `：` を `:` へ変換する
- [ ] 全角カンマ `，` を `,` へ変換する
- [ ] 全角セミコロン `；` を `;` へ変換する
- [ ] 各種ハイフン `−` / `–` / `—` / `ー` を `-` へ変換する
- [ ] 入力文字列を解析用に正規化する関数を実装する
- [ ] 正規化後も表示用の元文字列を保持する

### 1.3 書名エイリアス解析

- [ ] エイリアステーブルから書名候補を取得する
- [ ] 書名候補を文字数の長い順にソートする
- [ ] 最長一致で書名エイリアスを判定する
- [ ] `ヨハ一1:1` が `ヨハ一 + 1:1` と解釈されることを確認する
- [ ] 未登録の書名は解析失敗にする

### 1.4 単一聖句パーサー

- [ ] `{書名}{章}:{節}` を解析する
- [ ] `{書名} {章}:{節}` を解析する
- [ ] `{書名}　{章}:{節}` を解析する
- [ ] 全角数字・全角コロンを含む入力を解析する
- [ ] 章番号を数値として取得する
- [ ] 節番号を数値として取得する
- [ ] 元の入力文字列を `originalText` として保持する
- [ ] 書名エイリアスを `bookAlias` として保持する
- [ ] 書名番号を `bookNumber` として保持する
- [ ] 章番号を `chapter` として保持する
- [ ] 節情報を `parts` に格納する
- [ ] 変換開始位置 `startIndex` を保持する
- [ ] 変換終了位置 `endIndex` を保持する

### 1.5 不完全・非対応入力の除外

- [ ] `テトス1:` は変換しない
- [ ] `テトス1` は変換しない
- [ ] `テトス` は変換しない
- [ ] `テトス1章14節` は変換しない
- [ ] `テトス 1章14節` は変換しない
- [ ] 章節区切りがコロン以外の場合は変換しない

### 1.6 bible ID生成

- [ ] 書名番号を2桁にパディングする
- [ ] 章番号を3桁にパディングする
- [ ] 節番号を3桁にパディングする
- [ ] `{書名番号2桁}{章番号3桁}{節番号3桁}` 形式で生成する
- [ ] `テトス1:14` から `56001014` を生成する
- [ ] `詩編119:105` から `19119105` を生成する

### 1.7 URL生成

- [ ] デフォルトURLテンプレートを使ってURLを生成する
- [ ] `{{bible}}` を bible ID に置換する
- [ ] `{{wtlocale}}` を設定値に置換する
- [ ] `{{pub}}` を設定値に置換する
- [ ] `テトス1:14` 用URLを生成する

### 1.8 Markdownリンク生成

- [ ] 単一聖句参照をMarkdownリンク形式へ変換する
- [ ] 表示テキストには元の入力文字列を使用する
- [ ] `テトス1:14` を `[テトス1:14](url)` に変換する
- [ ] `テトス１：１４` を `[テトス１：１４](url)` に変換する
- [ ] URLには正規化済みの章節から生成した bible ID を使う

### 1.9 エディタ連携

- [ ] Obsidianエディタの入力変更イベントを購読する
- [ ] 現在フォーカス中の行を取得する
- [ ] 現在行のテキストを取得する
- [ ] 現在行だけを解析対象にする
- [ ] 変換対象範囲をエディタ位置へ変換する
- [ ] エディタ上の対象文字列をMarkdownリンクに置換する

### 1.10 デバウンス

- [ ] 入力変更時に既存タイマーをクリアする
- [ ] 設定値のデバウンス時間を参照する
- [ ] 初期値として1000msを使う
- [ ] 入力停止から1000ms後に解析を実行する
- [ ] 入力中の不完全な文字列は変換しない

### 1.11 有効・無効制御

- [ ] `enabled` 設定を参照する
- [ ] `enabled: true` の場合のみ自動変換する
- [ ] `enabled: false` の場合は入力監視しても変換しない

## Phase 2: 安全性

### 2.1 コードブロック除外

- [ ] 現在行がコードブロック内か判定する
- [ ] fenced code block の開始行を検出する
- [ ] fenced code block の終了行を検出する
- [ ] コードブロック内では解析を実行しない
- [ ] コードブロック内の `テトス1:14` を変換しない

### 2.2 インラインコード除外

- [ ] 現在行内のバッククォート範囲を検出する
- [ ] インラインコード範囲を除外範囲として保持する
- [ ] `` `テトス1:14` `` 内の参照を変換しない
- [ ] インラインコード外の参照だけ変換する

### 2.3 Markdownリンク除外

- [ ] Markdownリンク `[text](url)` の範囲を検出する
- [ ] リンクテキスト部分を除外する
- [ ] リンクURL部分を除外する
- [ ] `[テトス1:14](https://example.com)` を変換しない
- [ ] `[link](https://example.com/テトス1:14)` を変換しない

### 2.4 プレーンURL除外

- [ ] 現在行内の `http://` URLを検出する
- [ ] 現在行内の `https://` URLを検出する
- [ ] URL範囲を除外範囲として保持する
- [ ] `https://example.com/テトス1:14` を変換しない

### 2.5 Obsidian内部リンク除外

- [ ] `[[...]]` の範囲を検出する
- [ ] 内部リンク範囲を除外範囲として保持する
- [ ] `[[テトス1:14]]` を変換しない

### 2.6 角括弧・丸括弧内除外

- [ ] 単独角括弧 `[... ]` の範囲を検出する
- [ ] 丸括弧 `(...)` の範囲を検出する
- [ ] `[テトス1:14]` を変換しない
- [ ] `(テトス1:14)` を変換しない
- [ ] Markdownリンク検出と競合しないようにする

### 2.7 再変換防止

- [ ] 既存Markdownリンク化済みの聖句を検出する
- [ ] 既にリンク化された範囲は変換対象から除外する
- [ ] 自動変換後に同じ行を再解析しても二重リンクにならないことを確認する

### 2.8 除外範囲との照合

- [ ] 除外範囲リストを作成する
- [ ] 変換候補の範囲が除外範囲と重なるか判定する
- [ ] 除外範囲と重なる候補は変換しない
- [ ] 除外範囲外の候補は変換する
- [ ] 同一行に除外対象と変換対象が混在しても正しく処理する

### 2.9 カーソル位置補正

- [ ] 変換前のカーソル位置を取得する
- [ ] 変換対象がカーソルより前か判定する
- [ ] 変換前後の文字数差分を計算する
- [ ] 変換対象がカーソルより前の場合、差分ぶんカーソル位置を補正する
- [ ] 変換対象がカーソルより後ろの場合、カーソル位置を変更しない
- [ ] 複数置換時の累積差分を計算する

### 2.10 Undo制御

- [ ] 置換処理をCodeMirrorの単一トランザクションで実行する
- [ ] 同一行内の複数置換を1つのトランザクションにまとめる
- [ ] 1回のUndoで変換前文字列へ戻ることを確認する
- [ ] Undo後に不要な副作用が残らないことを確認する

### 2.11 エラー時の無音化

- [ ] 解析失敗時は何もしない
- [ ] 変換失敗時は何もしない
- [ ] 無効な聖句でも通知を出さない
- [ ] エラー表示を出さない
- [ ] ステータスバー表示を出さない
- [ ] 入力文字列をそのまま残す

## Phase 3: 複数指定

### 3.1 コンマ区切り解析

- [ ] 同じ章内のコンマ区切り節指定を解析する
- [ ] `啓示21:3,4` を解析する
- [ ] `啓示21:3, 4` を解析する
- [ ] `啓示21:3,4,5` を解析する
- [ ] `啓示21:3,5,7` を解析する
- [ ] 各節を `ScripturePart` として格納する
- [ ] 各パートの元文字列を保持する

### 3.2 コンマ後空白保持

- [ ] コンマ後の空白を検出する
- [ ] 変換後もコンマ後の空白を保持する
- [ ] `啓示21:3,4` を空白なしで出力する
- [ ] `啓示21:3, 4` を空白ありで出力する
- [ ] `啓示21:3, 5, 7` を空白ありで出力する

### 3.3 複数節リンク生成

- [ ] 最初のリンク表示に書名・章・節を含める
- [ ] 2つ目以降のリンク表示は入力された節指定部分のみとする
- [ ] 各節ごとに bible ID を生成する
- [ ] 各節ごとにURLを生成する
- [ ] `啓示21:3,4` を `[啓示21:3](url),[4](url)` に変換する
- [ ] `啓示21:3, 4` を `[啓示21:3](url), [4](url)` に変換する

### 3.4 範囲指定解析

- [ ] `開始節-終了節` の範囲指定を解析する
- [ ] 正規化後のハイフンで範囲を判定する
- [ ] 範囲開始節を `startVerse` に格納する
- [ ] 範囲終了節を `endVerse` に格納する
- [ ] 範囲指定の元文字列を保持する
- [ ] `啓示21:3-5` を解析する

### 3.5 範囲指定リンク生成

- [ ] 範囲指定のリンク先は開始節にする
- [ ] 範囲指定の表示テキストは元文字列を保持する
- [ ] `啓示21:3-5` を `[啓示21:3-5](...bible=66021003...)` に変換する

### 3.6 複合指定解析

- [ ] 節と範囲が混在する指定を解析する
- [ ] `啓示21:3-5,7,9-11` を解析する
- [ ] `啓示21:3-5, 7, 9-11` を解析する
- [ ] 各パートの開始節を取得する
- [ ] 各パートの終了節を必要に応じて取得する
- [ ] 各パートの元文字列を保持する

### 3.7 複合指定リンク生成

- [ ] 複合指定の各パートを個別リンク化する
- [ ] 範囲パートは開始節をリンク先にする
- [ ] 単一節パートはその節をリンク先にする
- [ ] `啓示21:3-5,7,9-11` を期待形式へ変換する
- [ ] `啓示21:3-5, 7, 9-11` を空白保持して変換する

### 3.8 章またぎ指定の除外

- [ ] コンマ後に `章:節` が現れた場合は章またぎとして判定する
- [ ] 範囲終了側に `章:節` が現れた場合は章またぎとして判定する
- [ ] `啓示21:3,22:1` は全体を変換しない
- [ ] `啓示21:3-22:1` は全体を変換しない
- [ ] `22:1` を節指定として誤変換しない

### 3.9 セミコロン区切り

- [ ] セミコロン `;` 区切りを検出する
- [ ] 全角セミコロン `；` を正規化して扱う
- [ ] セミコロン前後を独立した参照候補として解析する
- [ ] セミコロン区切り後に書名がある場合は変換する
- [ ] `創 24:59; 出 3:2` を両方変換する
- [ ] セミコロン区切り後に書名がない場合は変換しない
- [ ] `創 24:59; 35:8` の `35:8` は変換しない
- [ ] 書名省略継承を実装しない

### 3.10 行内複数箇所変換

- [ ] 1行内の複数聖句参照を検出する
- [ ] 通常行内の複数参照を変換する
- [ ] リスト行内の参照を変換する
- [ ] 番号付きリスト行内の参照を変換する
- [ ] 引用行内の参照を変換する
- [ ] Markdownテーブル行内の参照を変換する
- [ ] 同一行内の複数置換でインデックスずれが起きないようにする

## Phase 4: 検証・設定UI

### 4.1 章節データ

- [ ] デフォルト章節データ `verseMap` を静的データとして定義する
- [ ] 書名番号をキーとして章節データを参照する
- [ ] 章番号をキーとして最大節数を参照する

### 4.2 単一節検証

- [ ] 書名番号が `verseMap` に存在するか検証する
- [ ] 章番号が対象書に存在するか検証する
- [ ] 節番号が対象章に存在するか検証する
- [ ] 節番号が最大節数以内か検証する
- [ ] `テトス9:99` を無効として扱う
- [ ] `啓示21:999` を無効として扱う
- [ ] `啓示99:1` を無効として扱う

### 4.3 範囲指定検証

- [ ] 範囲開始節が存在するか検証する
- [ ] 範囲終了節が存在するか検証する
- [ ] 範囲終了節が最大節数以内か検証する
- [ ] 開始節 <= 終了節か検証する
- [ ] `啓示21:5-3` を無効として扱う
- [ ] 無効な範囲指定は変換しない

### 4.4 複数指定検証

- [ ] 複数指定内のすべての単一節を検証する
- [ ] 複数指定内のすべての範囲指定を検証する
- [ ] 1つでも無効なパートがある場合は全体を変換しない
- [ ] 有効な複数指定のみ変換する

### 4.5 基本設定UI

- [ ] 設定タブを作成する
- [ ] プラグイン有効/無効のトグルを追加する
- [ ] デバウンス時間の入力欄を追加する
- [ ] `wtlocale` の入力欄を追加する
- [ ] `pub` の入力欄を追加する
- [ ] URLテンプレートの入力欄を追加する
- [ ] 設定変更時に保存する
- [ ] 保存後の設定を自動変換処理に反映する

### 4.6 エイリアス設定UI

- [ ] 書名エイリアス一覧を表示する
- [ ] エイリアス追加UIを作成する
- [ ] エイリアス編集UIを作成する
- [ ] エイリアス削除UIを作成する
- [ ] 書名番号を指定できるようにする
- [ ] エイリアス変更後に最長一致用ソートへ反映する
- [ ] エイリアスJSONインポートを実装する
- [ ] エイリアスJSONエクスポートを実装する
- [ ] エイリアスをデフォルトに戻すボタンを追加する
- [ ] 不正なエイリアスJSONでは設定を更新しない

章節データ設定UIとテスト入力欄は実装対象外とする。

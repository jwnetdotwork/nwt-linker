## 3. Phase 2 前半: 変換除外範囲の検出

@docs/todo.md の以下を実装して。

- 2.1 コードブロック除外
- 2.2 インラインコード除外
- 2.3 Markdownリンク除外
- 2.4 プレーンURL除外
- 2.5 Obsidian内部リンク除外
- 2.6 角括弧・丸括弧内除外
- 2.8 除外範囲との照合

要件:
- 行内の除外範囲を検出する関数を実装する
- 変換候補と除外範囲の重なり判定を実装する
- コードブロック判定は現在行がブロック内か判定できるようにする
- 既存の変換処理に除外判定を組み込む

完了条件:
以下は変換されない。

- `[テトス1:14](url)`
- `[link](https://example.com/テトス1:14)`
- `https://example.com/テトス1:14`
- `` `テトス1:14` ``
- `[[テトス1:14]]`
- `[テトス1:14]`
- `(テトス1:14)`
- コードブロック内の `テトス1:14`

ただし、同じ行の除外範囲外にある `テトス1:14` は変換される。
